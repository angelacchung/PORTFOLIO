Mobile App - UX Audit

UX/UI

ROLE

Research, Visual Design, Data analysis, UX analysis

DURATION

July - August 2019

TOOLS

July - August 2019

SUMMARY

I had the opportunity to lead a UX audit of a client‚Äôs mobile app which focused on women‚Äôs health. I analyzed the app‚Äôs google analytics data, the UX cam recordings, researched their competitors and made recommendations on how they could improve their app. The client was particularly looking for ways on how to increase their user retention.

PROBLEM

Along with some UI and UX issues, the client expressed that their user retention was not as high as they wanted it to be. They were also seeing that there were high drop off rates on some of their screens. After this UX audit, the client applied some of the recommendations and increased their user retention rate from 30% to 50%.

SOLUTION

I completed this audit in the following phases: Data analysis, UX evaluation, Competitive Analysis, Recommendations and lastly, I showed the client a mockup design of what their app could look like.

In the beginning

DATA ANALYSIS

By collaborating with a fellow designer, I analyzed the behavior flow report as well as the UX Cam recordings of the users. The behavior flow report allowed us to see where users were dropping off from the app, which we took note of in order to later find the answer as to why. The UX Cam recordings gave us some insights into understanding how users were behaving on the app.

We saw that users were dropping off the most from the following screens:

  • Articles screen (home screen)
  • Signup screen (onboarding)
  • Chat with doctor
  • Chat standby screen

UX CAM ANALYSIS

My next step was to watch some recordings from UX Cam and see how users behave. A few observations and UX issues that I found were:

In the chatroom, users were tapping on the doctor‚Äôs profile picture.

Doctors often have to ask their users to elaborate on their medical issue.

Adding a dosage for medicine is confusing. Here, UX Cam shows that the user is rage tapping the screen.

Analyzing for UX issues

For phase 2, I analyzed for UX and UI issues on both Android and iOs devices. I found that:

Blog posts may be irrelevant to users‚Äô needs and interests. As a result, it‚Äôs hard for the user to identify the content they would want to read.

There is no clear indication that the user is being onboarded to chatting with a doctor.

Medication: If a user puts in a medication that is not from the drop down list (ex: Advil), it looks like it has been added to the list but the button turns into a loading animation and just keeps looping.

‚ÄòConnecting you with a doctor‚Äô popup can be exited but it takes you out of the queue which is unexpected.

Lack of giving the user a sense of progress being made

OTHER RECOMMENDATIONS

  • Let users choose if they want to chat with a doctor right away once they register
  • Give users the ability to create a medical profile for themselves
  • Ability to let users stay in queue when they exit the popup
  • Let users access the chat history, but not open the chat logs if they are not a pro member
